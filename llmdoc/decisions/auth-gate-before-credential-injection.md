# 虚拟 SK 门是纯附加层，位于凭证强制覆盖之前

鉴权门加在请求处理器入口早期（401 先于 404 未知路由判断），完全不碰下游凭证注入。

**Why:** 代理本来就用真实上游 key 强制覆盖客户端 key（`buildUpstreamHeaders`），下游无视客户端鉴权。所以边缘鉴权只能作为前置准入门、与凭证注入解耦。被否决方案：把鉴权塞进凭证注入路径——会把「边缘准入」和「上游凭证」两件事耦合。401 先于 404 是为了不泄漏路由是否存在（防路径探测）。

**How to apply:** 改鉴权只动 `authorizeVirtualKey` 和处理器入口的判断顺序，别动 `buildUpstreamHeaders`。
